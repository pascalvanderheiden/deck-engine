# Project Context

- **Owner:** Ali Soliman
- **Project:** DECKIO / deck-engine ŌĆö a presentation engine and scaffolder for creating beautiful, Copilot-powered slide decks
- **Stack:** Node.js monorepo (npm workspaces), Vite, HTML/CSS/JS, packages: `@deckio/deck-engine` and `@deckio/create-deck-project`
- **Created:** 2026-03-13

## Core Context

These are summarized key learnings from early sessions (2026-03-13 ŌåÆ 2026-03-14) that set the foundation:

1. **Dev Workflow + Infrastructure:** Phase 1 complete ŌĆö root scripts added (dev/build/test/lint), Vitest workspace tests, prepublishOnly gate on engine tests. `capture-screen.mjs` cross-platform. Export logic consolidated. Foundation for downstream tooling work.

2. **Browser/Node Boundary:** Critical lesson: never re-export Node.js-only modules from main browser entry (`index.js`). Dev server hides mistakes; Rollup production build enforces the boundary. Engine `index.js` is browser-safe; Node.js utilities live in `vite.js` and `themes/theme-loader.js`. Enforced at exports map level.

3. **Scaffolder Best Practices:** Two critical bugs taught lasting lessons: (1) Never hardcode dependency versions in templates ŌĆö always read from source `package.json` at scaffold time. For local dev, use `file:` protocol refs. (2) Unit tests are insufficient ŌĆö full E2E gate is required: scaffold ŌåÆ npm install ŌåÆ dev server ŌåÆ production build. Dev mode masks Node.js module boundary violations.

4. **Theme Architecture + Tailwind Integration:** Installed Tailwind CSS v4, created `themes/` directory with pluggable CSS-only theme system. Vite plugin exports `deckPlugin()`, `deckPlugins()` (combo), `tailwindPlugin()`. Theme loader provides `resolveTheme()` and `getAvailableThemes()`. Each theme = CSS file with `@import "tailwindcss"`, `:root` properties, and `@theme inline` bridge. No runtime JS involvement.

5. **shadcn/ui + Design System Integration:** Added opt-in `designSystem` field (separate from `theme` axis). When enabled: `components.json`, `jsconfig.json`, `@/*` alias, `src/lib/utils.js`, `ThemeProvider`, `.dark` mode support. Pre-generate files instead of running interactive `npx shadcn init`. All scaffolded artifacts generated deterministically (not interactive CLI dependencies).

6. **File Pre-generation Strategy:** Proven effective across multiple areas ŌĆö shadcn artifacts, `.vscode/mcp.json`, `components.json` ŌĆö makes scaffolded projects deterministic and faster. Pattern extends to theme creator design (next).
