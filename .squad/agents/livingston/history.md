- **shadcn/ui Compatibility Fix (2026-03-14):** Wrapped all engine CSS in `@layer` directives to prevent cascade conflicts with Tailwind v4 / shadcn/ui. Root cause: engine's `global.css` was entirely unlayered. In the CSS cascade, unlayered styles ALWAYS beat `@layer`'d styles regardless of specificity. Tailwind v4 wraps utilities in `@layer utilities`, so the engine's `* { margin: 0; padding: 0 }` would override every Tailwind margin/padding utility, breaking all shadcn components. Fix: declared `@layer theme, base, components, utilities;` at top of global.css (imported first, sets canonical order), put `:root` tokens + reset in `@layer base`, engine component styles (.slide, .deck, .deck-ty-*, etc.) in `@layer components`. Theme CSS `:root` blocks remain unlayered (highest priority, correctly overrides base defaults). CSS Module components (Navigation, BottomBar) are scoped by hashed class names — no conflict with Tailwind. `data-theme` attribute and shadcn's CSS-variable approach are compatible since both read from the same `--background`, `--foreground`, etc. tokens. No changes to theme CSS files, Vite plugin, or SlideContext needed. 126 tests pass.
